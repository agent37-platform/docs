## Get started in three steps

Get the Agent 37 docs running locally, make a small content change, and prepare your workflow for ongoing updates.

### Step 1: Set up your local environment

<AccordionGroup>
  <Accordion icon="copy" title="Clone the repository">
    Clone this repository to your local machine so you can edit and preview the docs before you publish changes.

    If you need a refresher, follow GitHub's guide for [cloning a repository](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository).
  </Accordion>
  <Accordion icon="rectangle-terminal" title="Start the local preview">
    1. Install the Mintlify CLI: `npm i -g mint`
    2. Navigate to your docs directory and run `mint dev`
    3. Open `http://localhost:3000` to preview the site

    <Tip>Your preview updates automatically as you edit files.</Tip>
  </Accordion>
</AccordionGroup>

### Step 2: Make your first Agent 37 update

<AccordionGroup>
  <Accordion icon="palette" title="Review the site configuration">
    `docs.json` controls the site name, navigation, branding, navbar links, and other shared settings.

    1. Open `docs.json` in your editor.
    2. Review the `"name"`, `"colors"`, and navigation sections.
    3. Update any links or labels that should match the current Agent 37 product experience.
    4. Save and confirm the changes in your local preview.

    <Tip>Small config updates are a fast way to confirm your local preview is working.</Tip>
  </Accordion>
  <Accordion icon="file-pen" title="Edit a page">
    Start with a high-value page such as `index.mdx` or `channels/whats-app.mdx`.

    Add or revise content, then refresh your preview and check that the page reads clearly from a user perspective.
  </Accordion>
</AccordionGroup>

### Step 3: Go live

<Accordion icon="rocket" title="Publish your changes">
  1. Create a branch for your work.
  2. Commit and push your changes.
  3. Follow your team's normal review and deployment workflow.
</Accordion>
