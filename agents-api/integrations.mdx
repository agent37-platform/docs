Connect an agent to Gmail, Slack, Notion, and 1000+ more, every app in the [Composio catalog](https://composio.dev/toolkits) is supported by default. Connections route and meter through Agent37, so there are no integration keys for you to manage. Each connection is scoped to a single [instance](/agents-api/instances).

## Endpoints

| Action | Endpoint |
| --- | --- |
| Search the catalog | `GET /v1/instances/{id}/integrations?search=` |
| Start a connection | `POST /v1/instances/{id}/integrations/connect` |
| List connections | `GET /v1/instances/{id}/integrations/connections` |
| Disconnect | `DELETE /v1/instances/{id}/integrations/connections/{cid}` |

## How a connection works

`connect` returns a `redirect_url`. Open it for your end user, they authorize the app, and the connection turns `active`. Connecting is free; the agent using a connected app during a run draws the connected-app meter against your prepaid balance.

<Steps>
  <Step title="Start a connection">
    `POST /v1/instances/{id}/integrations/connect` with a `toolkit`. The response carries a `connection_id`, a `redirect_url`, and a `pending` status.

    <CodeGroup>
    ```bash curl
    curl -X POST https://api.agent37.com/v1/instances/inst_x7/integrations/connect \
      -H "Authorization: Bearer sk_live_..." \
      -H "Content-Type: application/json" \
      -d '{ "toolkit": "gmail" }'
    ```

    ```python python
    import requests

    H = {"Authorization": "Bearer sk_live_...", "Content-Type": "application/json"}
    conn = requests.post(
        "https://api.agent37.com/v1/instances/inst_x7/integrations/connect",
        headers=H,
        json={"toolkit": "gmail"},
    ).json()
    ```

    ```javascript node
    const H = {
      "Authorization": "Bearer sk_live_...",
      "Content-Type": "application/json",
    };
    const conn = await (await fetch(
      "https://api.agent37.com/v1/instances/inst_x7/integrations/connect",
      { method: "POST", headers: H, body: JSON.stringify({ toolkit: "gmail" }) },
    )).json();
    ```
    </CodeGroup>
  </Step>

  <Step title="Send the user to authorize">
    Open `redirect_url` for your end user. They sign in to the app and grant access. No keys touch your code.
  </Step>

  <Step title="Poll until the connection is active">
    List the instance's connections and read the status. When it leaves `pending`, the agent can use the app on its next run.

    ```bash curl
    curl https://api.agent37.com/v1/instances/inst_x7/integrations/connections \
      -H "Authorization: Bearer sk_live_..."
    ```
  </Step>
</Steps>

<Tabs>
  <Tab title="Request">
    <ParamField body="toolkit" type="string" required>
      The Composio toolkit to connect, such as `gmail`, `slack`, or `notion`. Search the catalog for the full set of identifiers.
    </ParamField>

    ```json
    POST /v1/instances/inst_x7/integrations/connect
    { "toolkit": "gmail" }
    ```
  </Tab>
  <Tab title="Response">
    <ResponseField name="connection_id" type="string">
      The connection's id, prefixed `conn_`. Use it to disconnect.
    </ResponseField>
    <ResponseField name="redirect_url" type="string">
      The authorization URL to open for your end user.
    </ResponseField>
    <ResponseField name="status" type="string">
      `pending` until the user authorizes, then `active`.
    </ResponseField>

    ```json
    {
      "connection_id": "conn_5a",
      "redirect_url": "https://auth...",
      "status": "pending"
    }
    ```
  </Tab>
</Tabs>

## Disconnect

Remove a connection with `DELETE /v1/instances/{id}/integrations/connections/{cid}`, passing the `connection_id` as `{cid}`.

```bash curl
curl -X DELETE \
  https://api.agent37.com/v1/instances/inst_x7/integrations/connections/conn_5a \
  -H "Authorization: Bearer sk_live_..."
```

<Warning>
  Disconnecting revokes the agent's access to that app on the instance. Any session that relied on the connection loses it immediately.
</Warning>

## Billing

Connecting is free. Usage draws the connected-app meter against your prepaid balance whenever an agent uses a connected app during a run. Rates live on the [pricing page](https://agent37.com/pricing); balance and top-ups are managed in the dashboard.
