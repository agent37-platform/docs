---
title: "Scheduled runs"
description: "Give an agent a prompt and a cron schedule, then pull the results when you want them."
---

A schedule fires a [response](/agents-api/messages) on its instance at the times you set, with no caller attached. There are no webhooks. You pull results when you want them.

A schedule lives on one instance and runs unattended. Each time it fires, it produces a run you can read back later.

## Endpoints

| Action | Endpoint |
| --- | --- |
| Create | `POST /v1/instances/{id}/schedules` |
| List on an instance | `GET /v1/instances/{id}/schedules` |
| Update | `PATCH /v1/schedules/{sid}` |
| Delete | `DELETE /v1/schedules/{sid}` |
| Pause / resume | `POST /v1/schedules/{sid}/pause` · `/resume` |
| Run now | `POST /v1/schedules/{sid}/run` |
| Read results | `GET /v1/schedules/{sid}/runs` |

## Create a schedule

Pass a `cron` expression and the `input` the agent runs each time it fires.

<ParamField body="cron" type="string" required>
  A cron expression in the standard five-field format (`min hour day month weekday`). `0 8 * * *` fires daily at 08:00.
</ParamField>

<ParamField body="input" type="string" required>
  The message or task the agent runs on each fire. Reference uploaded files by path, same as a normal message.
</ParamField>

<ParamField body="agent" type="string">
  For a multi-agent template, which agent each run uses. Defaults to the template's primary.
</ParamField>

<ParamField body="metadata" type="object">
  Up to 16 key/value pairs. Echoed back, never interpreted.
</ParamField>

## A run

A schedule fires a response, and `GET /v1/schedules/{sid}/runs` returns one entry per fire.

<ResponseField name="id" type="string">
  The id of this run.
</ResponseField>

<ResponseField name="response_id" type="string">
  The `resp_` id of the response this run produced. Fetch it with `GET /v1/responses/{id}`.
</ResponseField>

<ResponseField name="status" type="string">
  `in_progress`, then `completed`, `failed`, or `cancelled`.
</ResponseField>

<ResponseField name="started_at" type="integer">
  When the run began, in unix seconds.
</ResponseField>

<ResponseField name="finished_at" type="integer">
  When the run ended, in unix seconds. Absent while `in_progress`.
</ResponseField>

<ResponseField name="output_text" type="string">
  The agent's final answer for this run.
</ResponseField>

<ResponseField name="usage" type="object">
  Token counts and `cost_usd` for the run.
</ResponseField>

## Daily digest

Create a schedule that fires every morning, then read what it has produced.

<Tabs>
  <Tab title="Create">
    <CodeGroup>
    ```bash curl
    curl -X POST https://api.agent37.com/v1/instances/inst_x7/schedules \
      -H "Authorization: Bearer sk_live_..." \
      -H "Content-Type: application/json" \
      -d '{
        "cron": "0 8 * * *",
        "input": "Summarize overnight emails into a digest."
      }'
    # -> { "id": "sched_3" }
    ```

    ```javascript node
    const res = await fetch(
      "https://api.agent37.com/v1/instances/inst_x7/schedules",
      {
        method: "POST",
        headers: {
          "Authorization": "Bearer sk_live_...",
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          cron: "0 8 * * *",
          input: "Summarize overnight emails into a digest.",
        }),
      },
    );
    const { id } = await res.json(); // "sched_3"
    ```
    </CodeGroup>
  </Tab>

  <Tab title="Read results">
    <CodeGroup>
    ```bash curl
    curl https://api.agent37.com/v1/schedules/sched_3/runs \
      -H "Authorization: Bearer sk_live_..."
    ```

    ```json response
    {
      "data": [
        {
          "id": "run_18",
          "response_id": "resp_9f2a",
          "status": "completed",
          "started_at": 1748505600,
          "finished_at": 1748505714,
          "output_text": "3 emails need a reply today...",
          "usage": { "input_tokens": 1840, "output_tokens": 410, "cost_usd": 0.0061 }
        }
      ],
      "has_more": false,
      "next_cursor": null
    }
    ```
    </CodeGroup>
  </Tab>
</Tabs>

<Note>
  Runs is a [list endpoint](/agents-api/concepts): it returns `{ data, has_more, next_cursor }`. Page with `?cursor=` and `?limit=`.
</Note>

## Pause, resume, and run now

<AccordionGroup>
  <Accordion title="Pause and resume" icon="player-pause">
    `POST /v1/schedules/{sid}/pause` stops a schedule from firing without deleting it; `POST /v1/schedules/{sid}/resume` starts it again on the same cron. Past runs stay readable while paused.
  </Accordion>
  <Accordion title="Run now" icon="player-play">
    `POST /v1/schedules/{sid}/run` fires the schedule immediately, outside its cron, and records a run like any other. Use it to test a new schedule without waiting for the next tick.
  </Accordion>
</AccordionGroup>

<Warning>
  `DELETE /v1/schedules/{sid}` removes the schedule. Pause instead if you only need to stop it temporarily.
</Warning>
