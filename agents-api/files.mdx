title: "Files"
description: "Upload a file for the agent to work on, and download the files it produces, straight from the instance URL."

Files live on the instance's disk, in the agent's workspace. A file's absolute path is its identity: there are no file ids. You upload with one call, attach the returned path to [a message](/agents-api/chat), and download anything the agent produces by path.

The base URL is your instance URL, `https://{instanceId}.agent37.app`, with the same `sk_live_` Bearer as every other call. This page uses `https://ab12cd34ef.agent37.app`.

## Upload a file

`POST /v1/files` takes a multipart body with exactly one file in a `file` field and writes it into the agent's workspace, under `uploads/` with a short random prefix so repeated uploads of the same name don't overwrite each other. The response carries the absolute `path` you attach to a turn.

<ParamField body="file" type="file" required>
  The file, as a multipart form field named `file`. One file per request; a missing or extra file returns `400 validation_error`.
</ParamField>

<CodeGroup>
```bash curl
curl -X POST https://ab12cd34ef.agent37.app/v1/files \
  -H "Authorization: Bearer sk_live_..." \
  -F "file=@leads.csv"
```

```python python
import requests

r = requests.post(
    "https://ab12cd34ef.agent37.app/v1/files",
    headers={"Authorization": "Bearer sk_live_..."},
    files={"file": open("leads.csv", "rb")},
).json()
```

```javascript node
import fs from "node:fs";

const form = new FormData();
form.append("file", new Blob([await fs.promises.readFile("leads.csv")]), "leads.csv");
const r = await (await fetch("https://ab12cd34ef.agent37.app/v1/files", {
  method: "POST",
  headers: { Authorization: "Bearer sk_live_..." },
  body: form,
})).json();
```

```json response
{
  "path": "/home/node/uploads/3f2a1b9c-leads.csv",
  "filename": "leads.csv",
  "bytes": 18244
}
```
</CodeGroup>

<ResponseField name="path" type="string">
  The file's absolute path on the instance. Pass it in `files` on [`POST /v1/responses`](/agents-api/chat).
</ResponseField>
<ResponseField name="filename" type="string">
  The original name you uploaded, without the storage prefix.
</ResponseField>
<ResponseField name="bytes" type="integer">
  The stored size in bytes.
</ResponseField>

<Note>
  The 2 MB request-body cap applies to JSON bodies only; multipart uploads are exempt. Uploads are kept until you delete them from the instance's disk yourself — there is no garbage collection.
</Note>

## Attach it to a turn

Pass the returned `path` in the `files` array on [`POST /v1/responses`](/agents-api/chat). The gateway appends the paths to your input and the agent reads them from disk:

```bash curl
curl https://ab12cd34ef.agent37.app/v1/responses \
  -H "Authorization: Bearer sk_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "input": "Summarize the attached spreadsheet.",
    "files": ["/home/node/uploads/3f2a1b9c-leads.csv"]
  }'
```

Each entry must name an existing file on the instance, or the call returns `400 validation_error` with the hint to upload it first.

## Download a file

`GET /v1/files/content?path=…` streams a file off the instance — typically one the agent told you it wrote. Any size; the 512 KB [exec](/agents-api/exec) output cap does not apply here.

<CodeGroup>
```bash curl
curl -G https://ab12cd34ef.agent37.app/v1/files/content \
  -H "Authorization: Bearer sk_live_..." \
  --data-urlencode "path=~/reports/ev-makers-memo.md" \
  -o ev-makers-memo.md
```

```python python
import requests

r = requests.get(
    "https://ab12cd34ef.agent37.app/v1/files/content",
    headers={"Authorization": "Bearer sk_live_..."},
    params={"path": "~/reports/ev-makers-memo.md"},
)
open("ev-makers-memo.md", "wb").write(r.content)
```

```javascript node
import fs from "node:fs";

const res = await fetch(
  "https://ab12cd34ef.agent37.app/v1/files/content?" +
    new URLSearchParams({ path: "~/reports/ev-makers-memo.md" }),
  { headers: { Authorization: "Bearer sk_live_..." } },
);
await fs.promises.writeFile("ev-makers-memo.md", Buffer.from(await res.arrayBuffer()));
```
</CodeGroup>

`path` accepts absolute paths and `~/` paths (the agent's home). No file at the path returns `404 file_not_found`; a path that is not a regular file, or a missing `path` parameter, returns `400 validation_error`.

## The loop

Upload, attach, fetch what the agent made:

1. `POST /v1/files` with the input file; keep the returned `path`.
2. `POST /v1/responses` with your `input` and the path in `files`.
3. When the agent replies that it wrote a file, `GET /v1/files/content?path=…` to fetch it.
