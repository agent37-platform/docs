A template is a prebuilt image that decides which agents an instance runs. Pass `template` when you create an [instance](/agents-api/instances), or omit it for the default.

## Available templates

| Template | Agents installed | Availability |
| --- | --- | --- |
| `hermes` | Hermes, general agent: browser, code, files. | Default, available today |
| `claude-code` | Claude Code, for software work. | Coming soon |
| `codex` | Codex, for software work. | Coming soon |
| `openclaw` | OpenClaw. | Coming soon |
| `all` | Hermes, Claude Code, Codex, and OpenClaw together. | Coming soon |

```bash curl
curl -X POST https://api.agent37.com/v1/instances \
  -H "Authorization: Bearer sk_live_..." \
  -H "Content-Type: application/json" \
  -d '{ "template": "hermes" }'
```

Omit `template` and you get `hermes`.

<Note>
  **Multi-agent templates.** When a template installs more than one agent (like `all`), a session runs whichever you choose with the `agent` field on its [first message](/agents-api/messages). The choice is fixed for the session's life. Single-agent templates ignore it.
</Note>

## Custom templates

<Info>
  Custom templates are in preview, and for now they start from a **public** image you host. Build an image `FROM` our base, push it to a public registry you already use, then register it with us and create instances from it. Private images and pushing straight to our own registry are coming soon.
</Info>

A custom template is your own image, built on our base and registered with us under a name you choose. Use it to preinstall [skills, plugins, and packages](/agents-api/skills#preinstall-with-a-custom-image) so every instance you create from it starts ready.

<Steps>
  <Step title="Build on our base">
    Write a Dockerfile that starts `FROM ghcr.io/agent37/base:latest` and layer your skills, plugins, and packages on top. The runtime, gateway, terminal, desktop, file browser, and managed auth all come from the base and keep working; the container has passwordless `sudo`, so you can install anything. See [Skills & plugins](/agents-api/skills#preinstall-with-a-custom-image) for the full Dockerfile.

    ```bash
    docker build --platform=linux/amd64 -t my-agent:v1 .
    ```

    Build for `linux/amd64` even on an Apple Silicon Mac. Instances run on amd64, and an arm64 image will fail to start.
  </Step>

  <Step title="Push it to a public registry">
    Push the image to any registry you already use, such as Docker Hub or GitHub Container Registry, and make sure it is public so we can pull it. You log in to your own registry the usual way; we never see your registry password.

    ```bash
    docker tag my-agent:v1 ghcr.io/your-name/my-agent:v1
    docker push ghcr.io/your-name/my-agent:v1
    ```
  </Step>

  <Step title="Register it with us">
    Tell us the image to pull and a name to call it. The template moves from `pending` to `ready`, or `failed` with a reason if we cannot pull it (a wrong reference, a missing tag, or a private image).

    ```bash curl
    curl -X POST https://api.agent37.com/v1/templates \
      -H "Authorization: Bearer sk_live_..." \
      -H "Content-Type: application/json" \
      -d '{ "name": "my-agent", "image": "ghcr.io/your-name/my-agent:v1" }'
    ```
  </Step>

  <Step title="Create instances from it">
    Once the template is `ready`, pass the name you registered (or its `tmpl_` id) as `template` when you create an [instance](/agents-api/instances).

    ```bash curl
    curl -X POST https://api.agent37.com/v1/instances \
      -H "Authorization: Bearer sk_live_..." \
      -H "Content-Type: application/json" \
      -d '{ "template": "my-agent" }'
    ```
  </Step>
</Steps>

### Manage templates

| Action | Endpoint |
| --- | --- |
| List built-in and custom templates | `GET /v1/templates` |
| Get one, to poll status | `GET /v1/templates/{id}` |
| Register an image you host | `POST /v1/templates` |
| Delete a custom template | `DELETE /v1/templates/{id}` |

<ResponseField name="id" type="string">
  The template id, prefixed `tmpl_`.
</ResponseField>
<ResponseField name="name" type="string">
  The name you chose, and what you pass as `template` when creating an instance, e.g. `my-agent`.
</ResponseField>
<ResponseField name="image" type="string">
  The image we pull, e.g. `ghcr.io/your-name/my-agent:v1`. Pin a tag; do not rely on `latest`.
</ResponseField>
<ResponseField name="status" type="string">
  `pending` while we pull and check the image, then `ready`, or `failed` with a reason if we cannot pull it.
</ResponseField>

<Tip>
  Building `FROM` our base is what gives an instance the managed agent, chat, terminal, desktop, and files. An image that is not built from our base still runs, but as a plain container without them.
</Tip>

<Note>
  **Coming soon.** Registering a **private** image (you give us a read-only pull token for your registry) and pushing an image **straight to our registry** at `registry.agent37.com`, so you can skip hosting it yourself. For now, use a public image.
</Note>
